./Crittografia_1

Questa rubrica sulla crittografia vuole essere un diario che raccoglie le tappe principali del mio percorso nello studio della crittografia, un’agenda utile per chiunque voglia iniziare a studiare da zero questa materia.

La storia della crittografia puo’ essere vista come una continua lotta tra crittografi e decrittografi. Nel corso della storia la crittografia e’ stata parte essenziale della vita di corte, ma anche di governi che la utilizzano per proteggere le loro comunicazioni ancora oggi. La crittografia e’ un elemento essenziale nelle conversazioni private, sia di singoli cittadini che di politici, ma anche di istituzioni e aziende.

Ad oggi, la crittografia viene adottata come standard nei protocolli che noi usiamo quotidianamente, basti pensare alla comunicazione online tra due computer. Un tempo pero’ non era cosi’, infatti re e governi dovevano usarla per proteggere le loro comunicazioni da lettori indesiderati.

Il primo esempio di crittografia che vedremo e’ il cifrario di Cesare. E’ un tipo di crittografia simmetrica, a sostituzione monoalfabetica. Ad ogni lettera del messaggio in chiaro ne corrisponde un’altra che si trova un certo numero di posizioni prima o dopo nell’alfabeto. Ad esempio la parola ‘ciao’ traslata secondo la chiave +1 diventa “djbp”. Ogni lettera del messaggio viene sostituita con la successiva.

Semplice implementazione in c del cifrario di Cesare.

#include <stdio.h>
#include <string.h>

void cifra_cesare(char *str, int chiave) {
    int k = chiave % 26;
    if (k < 0) k += 26;

    for (int i = 0; str[i] != '\0'; i++) {
        if (str[i] >= 'a' && str[i] <= 'z') {
            int offset = str[i] - 'a';
            int cifrato = (offset + k) % 26;
            str[i] = (char)(cifrato + 'a');
        }
    }
}

int main(void) {
    char messaggio[] = "ciao cesare"; 
    int chiave = 20;

    printf("Messaggio in chiaro: %s\n", messaggio);
    cifra_cesare(messaggio, chiave);
    printf("Messaggio cifrato:   %s\n", messaggio);

    return 0;
}
